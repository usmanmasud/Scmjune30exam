01 What is Git?

Git is a free, open-source distributed version control system (DVCS) created by Linus Torvalds in 2005. It allows multiple developers to track changes in source code, collaborate simultaneously, and revert to previous states of a project without losing work.

Unlike centralized systems, every Git user holds a full copy of the project history locally, making operations fast and enabling offline work.

02 Core Git Concepts

Repository (Repo)

A directory tracked by Git. Contains all project files and the full commit history inside a hidden .git/ folder.

Commit

A snapshot of changes at a point in time. Each commit has a unique SHA hash, an author, a timestamp, and a message.

Branch

An independent line of development. The default branch is usually called main. Branches let teams work in parallel.

Merge

Combines changes from one branch into another. Git attempts to merge automatically; conflicts are resolved manually.

Staging Area

Also called the index. A preparation zone where changes are reviewed before being committed to the history.

Clone

Creates a local copy of a remote repository, including all branches and the complete commit history.

04 What is GitHub?

GitHub is a cloud-based platform built on top of Git. It provides a web interface for hosting repositories, reviewing code, managing issues, and collaborating with teams worldwide. Founded in 2008 and acquired by Microsoft in 2018, GitHub hosts over 100 million developers.

While Git is the underlying tool that manages version history, GitHub adds collaboration features such as Pull Requests, code review, project boards, and GitHub Actions for CI/CD.

06 A Typical Git Workflow

  1. Initialise or Clone — Start a repo with git init or pull an existing one with git clone.
  2. Create a Branch — Isolate your feature or fix: git checkout -b feature/my-feature.
  3. Edit Files — Make changes to source code in your working directory.
  4. Stage Changes — Add modified files to the staging area: git add .
  5. Commit — Save a snapshot: git commit -m "feat: add login page".
  6. Push to Remote — Upload to GitHub: git push origin feature/my-feature.
  7. Open a Pull Request — Request a code review from teammates on GitHub.
  8. Merge — After approval, merge the branch into main.
